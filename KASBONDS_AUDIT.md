## Promotable to OpenSilver V1

### 1. MinimumBond → **Pattern 3.5/3.6 derivative: Oracle-Settled Escrow with Slash Distribution**

This is *not* the bilateral Escrow in the plan (3.5) and *not* the milestone Escrow (3.6). It's a third shape: oracle-released or timeout-slashed with a configurable multi-output slash distribution. It deserves its own slot.

Proposed Pattern 3.5b: **OracleEscrowWithSlash** (or fold into 3.12 Freelance/Payroll as the canonical implementation).

Generalisations needed before promotion:
- Drop hardcoded pubkeys (already done in parameterised variant).
- Make slash distribution shape generic: `(address, basis_points)[]` with sum == 10000. Currently 3 fixed outputs.
- Drop hardcoded `minerFee`; pass as ctor arg or derive via tx-level accounting.
- Add explicit `tx.outputs.length == N` check derived from distribution length (done for 3, not generic).
- Add a `dispute` entrypoint that requires multisig of oracle + buyer for early slash (planned but not implemented in current KasBonds).
- Cover all four paths with happy and failure tests on TN12.

### 2. X402Channel v4 → **Pattern 3.7 derivative: Stateful Payment Channel**

This is the cleanest example of `validateOutputState` in production. The pattern of `nonce + timeout` state with one-sided continuation (channel closes when remainder ≤ minerFee) is canonical.

Proposed: include as a worked **example** under Pattern 3.7 Streaming Payment, or promote to its own Pattern 3.7b **Payment Channel**.

Generalisations needed:
- Drop hardcoded facilitator pubkey; either ctor arg or wrap in `Ownable` (3.1) for facilitator rotation.
- Use KIP-20 covenant ID rather than implicit P2SH self-continuation (today's `validateOutputState` uses same-template hash; cov ID makes lineage explicit and OZ-style queryable).
- Add `expire(clientSig)` after timeout so client can sweep stale channels.
- Optional: per-recipient channel binding via `tx.outputs[0].scriptPubKey == byte[](new ScriptPubKeyP2PK(recipient))`.

### 3. Compiler vendoring strategy

KasBonds vendors `silverscript` under `vendor/silverscript`. OpenSilver should do the same in `upstream/silverscript` (already done) and lock to a known commit via either git submodule or a pinned tarball. This avoids depending on a moving master branch while Toccata stabilises.

## What KasBonds got right (preserve)

- **Hardcoded miner fee is honest and bounded.** OpenSilver pattern docs should keep this as the default with a "WHEN NOT TO USE THIS" callout if the fee market is volatile.
- **Multi-output value splits are computed inside the covenant**, not trusted from the spender. KasBonds slash split is `distributable * 50 / 100` etc. — that's the secure-by-construction default OpenSilver MUST keep.
- **Two-entrypoint shape (positive path + timeout path) is the canonical secure escrow.** No "withdraw any time" backdoors. Codify.

## What KasBonds taught us to fix in OpenSilver

- **Hardcoded constants force a recompile per deployment.** Every OpenSilver pattern ships parameterised first; constants version is generated by the CLI for testnet proofs only.
- **Stranded UTXOs from early test locks.** OpenSilver patterns should ship with a documented `recovery_address` ctor arg or be paired with a Pattern 3.9 (Dead Man's Switch) wrapper as a safety net for testnet experiments.
- **Distribution percentages baked into source = no governance.** Where slash/payout splits matter, encode them as state (1:1 transition) or as ctor args with their own audit checklist.
